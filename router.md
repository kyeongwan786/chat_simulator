# 📱 AI 연애 카톡 시뮬레이터 – 전체 페이지 구조 (11개)

---

## ✅ 1) `/welcome` — 온보딩/인트로 화면

### 역할
- 앱 첫 진입
- 감성 문구 + 앱 목적 소개
- “시작하기” → `/login` 또는 `/home` 이동

### 포함 요소
- 앱 로고/타이틀
- 한 줄 슬로건
- 시작하기 버튼 1개

### 연결
- welcome → login  
- welcome → home

---

## ✅ 2) `/login` 또는 `/profile_setup`
(MVP에서는 생략 가능 / 익명 로그인 OK)

### 역할
- 사용자 식별(ID 발급)
- 기본 프로필 입력(선택)

### 포함 요소
- 구글 로그인 버튼  
  또는  
- 닉네임 입력  
- 성별/연애 스타일 선택(선택)

### 연결
- login → home

---

## ✅ 3) `/home` — 메인 허브

### 역할
- 다른 기능으로 이동하는 대시보드

### 포함 요소
- 연애 카톡 시뮬레이터 시작
- 실전 카톡 분석하기
- 최근 세션 이어하기 (선택)
- 추천 시나리오 (선택)

### 연결
- home → scenario_list  
- home → real_chat  
- home → simulator (이어서 진행 시)

---

## ✅ 4) `/scenario_list` — 시나리오 선택

### 역할
다양한 연애 상황 선택:
- 첫 카톡  
- 썸 시작  
- 고백 직전  
- 권태기  
- 싸운 후 화해 등

### 포함 요소
- 시나리오 카드 리스트  
  - 제목  
  - 설명  
  - 난이도  
  - 턴 수

### 연결
- scenario_list → character_select

---

## ✅ 5) `/character_select` — 캐릭터 스타일 선택

### 역할
- 캐릭터 선택(남/여/스타일)
- 이미지(선택)
- 성격 태그 선택

### 포함 요소
- 캐릭터 카드
- 성격 태그(다정/쿨/장난/논리적)
- “선택하기” 버튼

### 연결
- character_select → simulator

---

## ✅ 6) `/simulator` — 메신저 UI (핵심 페이지)

### 역할
- 유저 입력 → AI 답변
- Firestore에 메시지 저장
- 호감도/관계 변화 계산

### 포함 요소
- 상단: 캐릭터 이름 / 상태
- 본문: 말풍선 리스트(ListView)
- 하단: 입력창 + 전송 버튼
- (선택) 코치 버튼

### 연결
- simulator → result (엔딩 도달 시)

---

## ✅ 7) `/result` — 엔딩 결과 + 코치 리포트

### 역할
- 10~15턴 후 엔딩
- BAD / NORMAL / GOOD / PERFECT
- 메시지 영향 분석
- 개선 조언 제공

### 포함 요소
- 엔딩 타입
- 호감도 그래프
- 분석 코멘트
- 잘한 점 / 아쉬운 점
- 다시하기 버튼
- 다른 시나리오 버튼

### 연결
- result → scenario_list  
- result → home

---

## ✅ 8) `/real_chat` — 실전 카톡 붙여넣기 분석

### 역할
- 실제 카톡 복붙 → AI 분석
- 위험요소, 분위기, 추천 멘트 제공

### 포함 요소
- 텍스트 입력창
- 붙여넣기 버튼
- “분석하기” 버튼
- 분석 결과 섹션  
  - 분위기  
  - 주도권  
  - 호감도 흐름  
  - 위험 포인트  
  - 추천 답장 리스트

### 연결
- real_chat → home  
- real_chat → simulator (“추천 답장으로 연습하기”)
