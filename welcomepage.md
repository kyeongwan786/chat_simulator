# 🎮 연애 상황·난이도 기반 카톡 시뮬레이터 – Welcome Page 디자인 기획서 (v1.0)

> **앱 컨셉 한 줄 요약**  
> “소개팅/연애 상황을 고르고, 난이도(Lv1~Lv5)에 따라 ‘말빨’로 깨는 카톡 시뮬레이션 게임”

Welcome Page는  
이 앱이 **“상황 + 난이도 기반 대화 공략 게임”**이라는 걸  
한눈에 인식시키는 **로비 화면**이다.

AI, 코칭, 연습 앱 느낌은 전부 숨기고,  
**“게임 시작” + “상황·난이도 공략”** 이미지만 남긴다.

---

## 1. 사용자 & 맥락 정의

### 1-1. 타겟 유저

- 소개팅/연애 카톡에서 **뭐라고 답해야 할지 막히는 사람**
- “말실수 한 번으로 흐름 꼬인 경험”이 있는 사람
- “나 진짜 말주변 없다”라고 자주 생각하는 사람
- 그냥 연애 게임처럼 **대화 감각을 올리고 싶은 사람**

### 1-2. 유저가 앱을 켜는 상황

- 소개팅 후, 애프터 카톡을 어떻게 해야 할지 몰라서 검색하다가 들어옴
- 썸 단계에서 답장 텀이 길어져서 어색해진 상태
- 상대가 무뚝뚝해서, “이걸 계속 이어가야 하나?” 고민하는 상태
- 이미 망한 대화 캡쳐를 보며 “레벨5였네…” 하는 자조 감성

## 2. Welcome Page의 역할 (딱 3가지)

1. **이 앱의 구조를 한 장면으로 설명**  
   - “상황을 고르고 → 난이도를 선택해서 → 대화를 플레이한다”

2. **압박이 아니라 ‘게임적 도전감’으로 접근**  
   - “너 말 못 해”가 아니라  
   - “Lv1부터 천천히 깨보자” 느낌

3. **다음 화면(`/scenario_list`)로 자연스럽게 유도**  
   - “상황 선택하러 간다”는 걸 직관적으로 이해하게 만들기

## 3. 전체 톤 & 무드

### 3-1. 키워드

- **상황 기반** (소개팅 / 썸 / 연인 / 재회 등)
- **레벨 기반 난이도** (Lv1 ~ Lv5)

### 3-2. 피해야 할 톤

- 자기계발 앱, 연애 컨설팅 앱 느낌
- “지금 시작하세요!”, “연애 코칭” 같은 광고 카피
- 과한 캐릭터/일러스트, 이모티콘 난무

## 4. 레이아웃 구조 (세로 기준 영역 분할)

SafeArea 기준 세로 화면을 **4개 영역**으로 나눈다.

[ A ] 상단 브랜드/설명 영역 (약 15~20%)

[ B ] 중앙 “상황 + 난이도” 요약 패널 (약 30~35%)
→ 이 앱이 어떤 구조인지 한 번에 보여주는 핵심

[ C ] 메인 CTA 영역 (약 20%)
→ “시작하기” 버튼 + 서브 안내 한 줄

[ D ] 하단 카테고리 키워드/난이도 요약 (약 15%)
→ “소개팅 • 썸 • 연인” + “Lv1~Lv5”


중요:  
**실제 대화 말풍선, 실제 카톡 UI는 여기서 보여주지 않는다.**  
웰컴은 “세계관/구조”만 보여준다.

## 7. 난이도(Lv) 개념 시각화 (웰컴에서의 표현)

웰컴 페이지에서는 **Lv를 간접적으로 암시**만 한다.

### 7-1. B 영역 – “상황+난이도 요약 패널”

중앙에 작은 패널 배치:

- 상단 텍스트:
  - **“예: ‘소개팅 후 첫 카톡’”**
- 그 아래 작은 라인:
  - `Lv1  |  Lv2  |  Lv3  |  Lv4  |  Lv5`

여기서:
- Lv1~Lv5 텍스트는 전부 동일한 스타일  
- 단, **Lv1~Lv5 뒤에 짧은 설명 한 줄**만 붙임

예:

- Lv1 – 상대도 호감 있는 편, 애프터가 거의 확정  
- Lv3 – 서로 호감은 있지만, 대화가 어색한 단계  
- Lv5 – 상대가 무뚝뚝하고, 관심도도 애매한 상황

→ 웰컴에서는 이 설명들을 **아주 작게**, 2줄 정도로만 보여준다.  
**“레벨에 따라 상황 난이도가 달라진다”는 개념만 인식시키면 됨.**

## 8. 레이아웃 상세 구조

### 8-1. A 영역 – 상단 브랜드/설명

- 좌측 상단에 앱명 또는 로고 텍스트 1줄
  - 예: `Level Chat` / `Lv Talk` / (이름 미정이면 그냥 비워둬도 됨)
- 바로 아래 작은 설명:
  - **“연애/소개팅 상황을 난이도별로 플레이하는 카톡 시뮬레이터”**

폰트:
- Title: 18~20, SemiBold  
- Description: 13~14, Regular

---

### 8-2. B 영역 – “상황 + 난이도 요약 패널”

카드 형태:

- 가로 폭: 화면 너비의 90%  
- 높이: 전체 높이의 약 30~35%  
- 배경: `#15161E` 정도의 딤  
- 라운드: 20~24px  

카드 안 구성:

1. 상단 작은 라벨:
   - `예: 소개팅 후 첫 카톡`
2. 중간 굵은 텍스트(설명):
   - **“같은 상황도, 레벨에 따라 난이도가 달라집니다.”**
3. 하단 “Lv1~Lv5 요약 라인”:

예시:

- `Lv1  편한 상대, 애프터 거의 확정`
- `Lv3  서로 호감 있지만, 어색한 분위기`
- `Lv5  무뚝뚝하고, 관심도도 애매한 상대`

→ 실제 UI는 ListTile/Row로  
  Lv1~Lv5 세 줄만 간단하게 요약해서 보여줌.

**주의:**  
여기서 채팅 버블, 메시지 텍스트는 보여주지 않는다.

---

### 8-3. C 영역 – CTA 버튼 + 안내

버튼:

- 텍스트: **“상황 고르고 시작하기”**
- Color: `#FF5C8A → #FF8FB1` 그라데이션
- Radius: 18~20
- Height: 54~56
- Width: 85~90%

아래 안내 문구:

- **“처음엔 Lv1부터, 익숙해지면 Lv5까지 도전해보세요.”**  
- Secondary Text Color, 13~14pt

---

### 8-4. D 영역 – 하단 카테고리 키워드

중앙 정렬 텍스트:

> **“소개팅  •  썸  •  연인  •  재회”**

폰트:
- 크기: 13~14
- 색상: `#AEB0BB`
- Letter spacing: 약간 넓게 (0.3~0.4 정도)

역할:
- “이 앱이 어떤 상황들을 다루는지”를 한 번에 보여주는 태그라인

## 9. 인터랙션 설계

- 페이지 진입:
  - 상단 텍스트 → 패널 → 버튼 순으로  
    **200ms 간격 페이드 + 8px 슬라이드 업**
- CTA 버튼:
  - 누를 때:
    - scale 0.97, 그림자 줄어듦
  - 부드럽게 복귀
- 배경:
  - 느린 줌인(배경 이미지 scale 1.0 → 1.03 정도, 10초 주기)  
  - 너무 티 안 나게, 감정만 유지

- ❌ 설명이 길고 텍스트 많은 화면

## 11. 최종 화면 요약 (실제 유저 관점)

앱을 켠 유저는:

1. 위쪽 헤드라인에서  
   → “아, 이건 상황·난이도에 따라 대화 공략하는 게임이구나” 느끼고

2. 중앙 카드에서  
   → “아, 소개팅 후 첫 카톡, 이런 상황들을 Lv1~Lv5로 나누는구나”를 이해하고

3. 버튼 “상황 고르고 시작하기”를 보면서  
   → “일단 상황부터 고르러 가자”라고 자연스럽게 `/scenario_list`로 이동한다.

이 시점까지:

- “AI/연습/코칭”이라는 단어는 한 번도 안 나온다.
- “게임적인 도전감 + 현실 연애 상황”만 머릿속에 남는다.
